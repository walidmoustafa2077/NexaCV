## 1. SCOPE
### 1.1. System Identification
This document applies to **NexaCV**, an AI-enhanced, web-based resume builder application.

### 1.2. System Overview
NexaCV is a Software-as-a-Service (SaaS) platform designed for fresh graduates, senior professionals, and career changers. It allows users to quickly generate professional-grade resumes by inputting raw data (Basic Info, Education, Work Experience) into a multi-step wizard. The system utilizes Large Language Models (GPT-4o-mini) to professionally rewrite work experiences, generate summaries, and suggest skills. The system operates on a pay-per-download and pay-per-refinement business model.

### 1.3. System Requirements Document Overview
This SRD defines the functional, non-functional, data, and interface requirements for NexaCV. It serves as the primary blueprint for the Next.js frontend and .NET Backend developers.

## 3. SYSTEM REQUIREMENTS

### 3.1. Required States and Modes
*   **Guest Mode:** User can browse template gallery.
*   **Authenticated Mode:** User has created an account and can enter data into the multi-step wizard.
*   **Processing Mode:** System is awaiting response from the AI API.
*   **Review/Edit Mode:** User is interacting with editable text boxes, regenerating sections.
*   **Checkout Mode:** System calculates final price (Base + Regenerations) and locks the document for payment.
*   **Fulfilled Mode:** Payment is successful; user can download PDF/Word formats.

### 3.2. System Functional Requirements

#### 3.2.1. User Authentication
*   **Req 3.2.1.1:** The system shall require users to create an account either before starting or immediately after selecting a template.
*   **Req 3.2.1.2:** Authentication shall be handled via JWT (JSON Web Tokens) generated by the .NET backend.

#### 3.2.2. Data Input (Multi-Step Wizard)
*   **Req 3.2.2.1 Core Data:** The system shall collect Required fields: Full Name, Phone, Email, Location, and Core Work Experience (Job Title, Company).
*   **Req 3.2.2.2 Optional Data:** The system shall allow users to input: Summary, Education, Skills, Links, Projects, Certifications, Languages, Awards, Volunteer Exp, Publications, and Interests.

#### 3.2.3. AI Generation & Behaviors
*   **Req 3.2.3.1 Experience Rewriting:** The AI shall rewrite user-provided Work Experience bullet points using professional, action-oriented language based on user prompts.
*   **Req 3.2.3.2 Skill Suggestion:** The system shall auto-suggest skills based on the user's Job Title, Work Experience, or Education.
*   **Req 3.2.3.3 Section Regeneration:** Users shall be able to click a "Regenerate" button on specific sections, providing a custom text prompt to guide the AI.
*   **Req 3.2.3.4 Regeneration Limits:** The system shall enforce a strict limit of **3 AI regenerations per specific section** to prevent abuse.

#### 3.2.4. Template & Output Support
*   **Req 3.2.4.1 Diversity:** The system shall launch with at least 3 distinct templates catering to different industries (e.g., Corporate, Creative, Modern Minimalist).
*   **Req 3.2.4.2 Export Formats:** The system shall generate the final resume as a PDF (Mandatory). If the selected template supports it, MS Word (DOCX) export shall be provided.

#### 3.2.5. Pricing & Checkout Logic
*   **Req 3.2.5.1 Base Pricing:** The base price of a resume shall start from **120 EGP**, variable based on the chosen premium template.
*   **Req 3.2.5.2 Regeneration Pricing:** Each AI regeneration triggered by the user shall add **~10 EGP** to the final checkout price.
*   **Req 3.2.5.3 Dynamic Cost Display:** The UI shall display a real-time running total of the cost as the user consumes regenerations.
*   **Req 3.2.5.4 Multi-Currency:** The system shall default to EGP for local IP addresses and USD for global users.

### 3.3. System External Interface Requirements
*   **OpenAI Interface:** The .NET Backend shall securely communicate with the GPT-4o-mini endpoint using API keys stored in GCP Secret Manager / Environment variables.
*   **Payment Interface:** The system shall redirect to Stripe (or a local gateway like Paymob for EGP) for secure checkout, utilizing Webhooks to confirm payment status before allowing document download.

### 3.4. System Internal Data Requirements
*   **Req 3.4.1 Database:** Relational data (Users, Transactions, Resume Metadata) shall be stored in a **PostgreSQL** database.
*   **Req 3.4.2 Document Storage:** Final generated PDFs and user-uploaded assets (if any) shall be saved securely on the **GCP VM Disk Drive** (mapped to a Docker volume).
*   **Req 3.4.3 Data Persistence:** User resume data shall be stored permanently to allow them to log back in, pay again, or update their resume in the future.

### 3.8. Security and Privacy Requirements
*   **Req 3.8.1 PII Protection:** Sensitive user data (Phone, Email, Location) shall be transmitted exclusively over HTTPS/TLS 1.2+.
*   **Req 3.8.2 Access Control:** A user shall only be able to view, edit, or download resumes associated with their own `user_id`.

### 3.9. System Environment Requirements
*   **Req 3.9.1 Cloud Infrastructure:** The system shall be hosted on a single Google Cloud Platform (GCP) Compute Engine Virtual Machine.
*   **Req 3.9.2 Containerization:** Both the Next.js frontend and .NET backend shall be containerized using Docker and orchestrated via Docker Compose on the VM.

### 3.10. Computer Resource Requirements (Scalability fill-in)
*   **Req 3.10.1 Concurrency:** The initial GCP VM (e.g., e2-medium or e2-standard-2) shall be configured to handle up to 100 concurrent active users seamlessly.
*   **Req 3.10.2 Connection Pooling:** The PostgreSQL database shall use connection pooling (e.g., PgBouncer or Entity Framework pooling) to prevent memory exhaustion on the VM.

### 3.11. System Quality Factors (Performance & Reliability)
*   **Req 3.11.1 Latency:** AI generation requests must return a response to the UI in **under 5 seconds**.
*   **Req 3.11.2 Graceful Degradation:** If the OpenAI API is down or times out, the system shall display a user-friendly dialog/banner stating: *"AI assistance is temporarily unavailable. You may continue to type your resume manually."*

### 3.12. Design and Construction Constraints (UI/UX)
*   **Req 3.12.1 Design Language:** The application shall utilize a **Modern Minimalist** UI framework (e.g., Tailwind CSS + Shadcn UI).
*   **Req 3.12.2 Form Flow:** The data entry process must be structured as a **Multi-step Wizard**, grouping related information (e.g., Step 1: Personal, Step 2: Work, Step 3: Education) rather than a single long scrolling page.

## 4. VERIFICATION PROVISIONS

### 4.1. Verification Methods
*   **Demonstration:** AI regeneration limits (max 3) and real-time pricing updates (+10 EGP) will be verified via UI demonstration.

*   **Inspection:** The final PDF output will be visually inspected to ensure it matches the HTML/CSS template selected by the user.
